# Life Memory Vault — 초보자 빠른 시작 가이드

> **한 줄 정리**: 텔레그램에 사진·링크·메모를 던지면, 시스템이 알아서 분류·정리·요약하고 나중에 찾아줍니다.

---

## 이 프로젝트가 뭔가요?

**Life Memory Vault**는 일상에서 스치는 모든 것을 **마찰 없이 기록**하고, 나중에 **쉽게 찾을 수 있게** 만드는 개인 기억 시스템입니다.

### 어떤 것을 할 수 있나요?

| 내가 하는 것 | 시스템이 해주는 것 |
|---|---|
| 링크를 텔레그램에 전송 | 페이지 내용을 한국어로 요약해서 옵시디언에 저장 |
| 유튜브 링크 전송 | 영상 자막을 분석해 한국어 영상 요약 생성 |
| 영수증·화이트보드 사진 전송 | 사진 속 텍스트를 OCR로 읽어 검색 가능하게 저장 |
| PDF 파일 전송 | 문서 내용을 텍스트로 추출해 검색 가능하게 저장 |
| "카페 다음주에 가야지" 메모 전송 | 날짜 보강(26.MM.DD 형식) + 맛집으로 자동 분류 |
| "와이퍼 오늘 교체했음" 전송 | 차량 유지보수로 자동 분류 |
| 나중에 "카페" 검색 | 모든 카페 메모를 관련도 순으로 보여줌 |
| 아무것도 안 해도 | 매주 회고 메시지가 자동으로 텔레그램으로 옴 |

## 시작하기 전에 알아야 할 것

### 봇이 두 개입니다

```
📥 캡처 봇 (@my_lifelog_memory_bot)
   → 저장 전용. 뭐든 여기로 보내면 무조건 저장됨.
   → 명령(정리/수리 등)은 매일 밤 23시에 처리됨.

🤖 운영 봇 (@songbonds_CCC_bot)
   → 검색·정리·요약을 "지금 바로" 처리.
   → 자연어로 말해도 됨: "카페 메모 찾아줘", "링크 요약해줘"
```

> **기억하세요**: 평소엔 캡처 봇만 쓰면 됩니다. 시스템이 알아서 정리해줍니다.

### 옵시디언이 결과를 보는 창입니다

정리된 메모는 **옵시디언(Obsidian)** 앱에서 볼 수 있습니다.
- 텔레그램으로 저장한 메모들이 자동으로 분류되어 폴더에 정리됩니다.
- 링크 요약, OCR 결과, 유튜브 요약이 모두 노트에 자동으로 채워집니다.

## 샘플 시나리오

### 시나리오 1: 유튜브 영상 나중에 볼게

> "GPT-4o 리뷰 영상 봤는데 나중에 다시 보고 싶다."

1. 유튜브 URL을 복사
2. 캡처 봇에 붙여넣기 전송
3. 봇이 "저장 완료" 답변
4. 다음 날 오전, 옵시디언에 가면:

```
> [!abstract] 영상 요약 — GPT-4o 완벽 분석: 실제로 얼마나 뛰어난가?
> 출처: https://youtu.be/... · Ai-Con Lab · 수집 26.06.14

이 영상은 OpenAI의 GPT-4o 모델을 실제 작업에 적용해보며 성능을 
분석한 콘텐츠입니다. 특히 멀티모달 능력과 한국어 처리 성능에 
초점을 맞추고 있습니다.

**핵심 포인트**
- GPT-4o는 이전 모델 대비 속도가 2배 빠르며 비용은 절반
- 한국어 이해력이 크게 향상되어 실용적 활용도 증가
- 멀티모달(이미지+텍스트 동시 처리)이 핵심 차별화 포인트
```

→ **영상을 안 봐도** 핵심 내용을 바로 파악할 수 있습니다.

---

### 시나리오 2: 좋은 글 링크 저장

> "개발 블로그 글 읽다가 나중에 다시 봐야 할 것 같아서."

1. 브라우저에서 URL 복사
2. 캡처 봇에 전송
3. 22:00에 시스템이 페이지 내용을 읽음
4. 23:00에 AI가 한국어 요약 생성
5. 옵시디언에서 확인:

```
> [!abstract] 웹 페이지 — React 18의 Concurrent Features 완벽 이해
> 출처: https://blog.example.com/... · Example Blog · 수집 26.06.14

React 18에서 도입된 동시성 기능(Concurrent Features)에 대한 
심층 분석 글입니다. startTransition, Suspense, useDeferredValue 
등 새로운 API의 실제 사용 사례와 성능 개선 효과를 다룹니다.

**핵심 포인트**
- startTransition으로 UI 반응성을 유지하면서 무거운 상태 업데이트 처리 가능
- Suspense가 데이터 페칭까지 확장되어 로딩 상태 관리가 단순해짐
- 기존 React 17 코드와 완전히 호환되어 점진적 마이그레이션 가능
```

→ **원문을 다시 읽지 않아도** 핵심을 바로 확인할 수 있습니다.

---

### 시나리오 3: 영수증·화이트보드 사진

> "회의 때 화이트보드에 적은 아이디어를 나중에 찾고 싶다."

1. 화이트보드 사진 찍기
2. 캡처 봇에 사진 전송
3. 봇이 "저장 완료"
4. 22:00에 OCR로 텍스트 자동 인식
5. 23:00에 AI가 내용을 한국어로 정리
6. 옵시디언 노트에 추가됨:

```
> [!abstract] 이미지 (OCR) — whiteboard-photo.jpg
> 파일: [[80_Assets/Originals/images/...]] · 추출 26.06.14 · 방법: ocr

회의에서 나온 신규 기능 아이디어 메모입니다.
사용자 온보딩 플로우 개선과 대시보드 개편이 주요 주제였습니다.

**핵심 포인트**
- 온보딩 3단계 → 2단계로 줄이기 (이탈률 감소 목표)
- 대시보드에 최근 활동 피드 추가
- 모바일 앱 알림 개인화 기능 요청
```

→ 나중에 "온보딩" 또는 "대시보드"로 검색하면 이 사진 메모가 뜹니다.

### 시나리오 4: 할 일 메모

> "다음 주 화요일까지 보고서 제출해야 하는데."

1. 캡처 봇에 전송: `"보고서 제출 다음주 화요일까지"`
2. 자동으로 `30_Actions/Tasks` 폴더에 task로 분류
3. "다음주 화요일"이 `26.06.17` 형식으로 자동 변환
4. 옵시디언 Tasks 폴더에서 날짜별로 정리됨

## 자동화 타임라인 (시스템이 알아서 하는 것들)

```
상시 (24시간)
  └─ 캡처 봇이 텔레그램 메시지 대기 중
     → 받으면 즉시 "저장 완료" 응답

5분마다
  └─ 명령 처리
     → /검색, /상태, /통계, /점검 요청에 자동 응답

매일 22:00
  ├─ 정리(Lint): 분류되지 않은 메모를 폴더에 배치, 날짜 보강
  ├─ URL 추출(Enrich): 링크·유튜브 URL에서 내용 추출
  └─ 미디어 추출: 이미지 OCR, PDF 텍스트 추출

매일 23:00
  └─ AI 한국어 요약
     → 추출된 웹페이지·유튜브·이미지·PDF 내용을 한국어로 요약
     → 옵시디언 노트에 요약 자동 추가

매주 1회 (설정된 시간)
  └─ 주간 회고 메시지 → 캡처 봇으로 자동 발송
```

> 즉, **저장하는 것만 내가 하면** 나머지는 시스템이 다 합니다.

## 자주 쓰는 명령어

### 캡처 봇 명령 (매일 밤 23시 처리)

| 명령 | 하는 것 |
|---|---|
| `/도움` 또는 `/help` | 도움말 보기 (즉시) |
| `/상태` 또는 `/status` | 현재 시스템 상태 (즉시) |
| `/검색 카페` 또는 `/seek 카페` | 카페 메모 찾기 (즉시) |
| `/통계` 또는 `/digest` | 저장 통계 보기 (약 5분) |
| `/점검` 또는 `/doctor` | 도구 설치 상태 확인 (약 5분) |
| `/정리` 또는 `/lint` | 분류 작업 요청 (23시 배치) |
| `/웹요약` 또는 `/enrich` | 링크 요약 요청 (23시 배치) |

│   └── Tasks/         ← 할 일 메모들

│   ├── Food/          ← 맛집·카페 경험
│   └── Travel/        ← 여행 기록

## 태그로 분류 힌트 주기

메모 안에 `#태그`를 넣으면 분류에 반영됩니다:

| 태그 | 효과 |
|---|---|
| `#task` 또는 `#할일` | 할 일로 분류 |
| `#아이디어` 또는 `#idea` | 아이디어로 분류 |
| `#음악` 또는 `#music` | 음악 기록으로 분류 |
| `#여행` 또는 `#trip` | 여행 기록으로 분류 |
| `#private` 또는 `#비공개` | 민감 기록 표시 (공개 위치로 옮기지 않음) |
| `#노요약` | 링크가 있어도 자동 요약 제외 |

태그가 없어도 내용을 분석해 자동 분류하니까, 태그는 **선택 사항**입니다.

## 분류가 틀렸을 때

시스템이 메모를 잘못 분류했다면:

**방법 1 — 운영 봇에 말하기 (가장 쉬움)**
```
"이 메모 idea야, task 아니야"
"블루보틀 노트를 맛집으로 옮겨줘"
```

**방법 2 — CLI로 직접**
```bash
python3 scripts/mem.py reclassify "파일경로.md" --type idea
```

같은 유형을 두 번 수정하면 **시스템이 학습**해서, 그 다음부터는 비슷한 메모를 자동으로 올바르게 분류합니다.

## 자주 묻는 질문

**Q: 캡처 봇에 보내면 바로 옵시디언에 보이나요?**
A: 원본은 즉시 저장됩니다. 하지만 분류(정리)는 매일 22:00에 자동으로 됩니다. 정리된 노트를 지금 바로 보고 싶으면 운영 봇에 "정리해줘"라고 하세요.

**Q: 링크 요약은 얼마나 걸리나요?**
A: 링크 내용 추출은 22:00에, AI 한국어 요약은 23:00에 처리됩니다. 지금 바로 보고 싶으면 운영 봇에 "링크 요약해줘"라고 하세요.

**Q: 사진을 보내면 원본은 어디 가나요?**
A: `80_Assets/Originals/images/` 폴더에 영구 보관됩니다. OCR 텍스트도 `80_Assets/Extracts/images/`에 따로 보관됩니다. 삭제되지 않습니다.

**Q: 같은 링크를 두 번 보내면 어떻게 되나요?**
A: 중복을 자동으로 감지해서 노트는 하나만 만들어집니다. 원본은 두 개 다 보관되지만 정리본은 하나만 생깁니다.

**Q: 스크린샷 속 한국어도 OCR이 되나요?**
A: 됩니다. tesseract가 한국어+영어 동시 인식을 지원합니다. 단, 글씨가 작거나 배경이 복잡하면 인식률이 떨어질 수 있습니다.

**Q: 23시 배치가 갑자기 멈춘 것 같아요.**
A: 캡처 봇으로 "⚠️ 예약 작업 실패" 알림이 왔다면 Claude Code 채널 터미널의 로그인이 풀린 것입니다. 그 터미널에서 재로그인하면 다음 배치부터 자동으로 다시 처리됩니다.

**Q: 음성 메모는 텍스트로 변환되나요?**
A: 아직 지원되지 않습니다. 음성 파일은 저장만 됩니다. 텍스트 전사 기능은 추후 업데이트 예정입니다.

# Life Memory Vault — 사용자 가이드

> 한 줄 요약: **그냥 던지세요. 정리는 시스템이 합니다.**
> 떠오르는 메모·링크·사진·PDF를 아무렇게나 저장하면, 자동으로 분류·정리되고, 나중에 쉽게 찾을 수 있습니다.

이 가이드는 "쓰는 사람"을 위한 설명서입니다. 설치·서버 운영은 [운영 가이드](operator-guide.md)를 보세요.

---

## 1. 언제 쓰면 되나요?

이런 게 머릿속에 스칠 때 **고민 없이** 던지세요:

- 나중에 볼 링크/도구 (예: 깃허브 리포, 써보고 싶은 서비스)
- 유튜브 영상 (나중에 한국어 요약 자동 생성)
- 할 일·약속·결정 (예: "매달 말일 송금", "6/16 점심 약속")
- 사고 싶은 것, 산 것
- 차량·집·가전 교체/정비 기록
- 맛집·카페, 여행, 음악
- 아이디어, 메모, 그냥 적어두고 싶은 것
- **사진·스크린샷** (안에 있는 텍스트가 자동으로 OCR 추출됨)
- **PDF 파일** (문서 내용이 자동으로 텍스트 추출됨)

**핵심 원칙: 저장할 때 분류를 고민하지 마세요.** 분류는 나중에 시스템이 자동으로 합니다.

## 2. 기억 남기기 (캡처)

### 방법 A — 텔레그램 (제일 쉬움, 추천)

**캡처 봇**(`@my_lifelog_memory_bot`)에게 메시지를 보내면 끝입니다.

- **글**: 그냥 입력해서 전송
- **링크 / 유튜브 URL**: URL 붙여넣기 → 자동으로 한국어 요약 생성됨
- **사진/스크린샷**: 그대로 전송 → 사진 속 텍스트가 OCR로 자동 추출됨
- **PDF 파일**: 그대로 전송 → 문서 내용이 자동으로 텍스트 추출됨
- **음성메모/동영상**: 그대로 전송 (파일로 저장, 요약은 추후 지원 예정)
- 보내면 봇이 **"저장 완료"** 로 답합니다.
- 명령이 헷갈리면 **`/도움`**(또는 `/help`)을 보내세요 — 한국어 명령표와 처리 시점을 안내합니다.

> **봇이 둘입니다 (권장 동선):**
> - 📥 **캡처 봇** (`@my_lifelog_memory_bot`) — **저장 전용**. 메모·링크·사진 전부 이 봇으로.
> - 🤖 **운영 봇** (`@songbongs_CCC_bot`) — **검색·정리·요약을 지금 바로** 자연어로 시키고 싶을 때. 예: "안 정리된 메모 정리해줘", "유튜브 링크들 요약해줘", "카페 메모 찾아줘".
>
> 캡처 봇의 `/정리`·`/수리` 등은 **매일 23시 배치**로 처리됩니다. 즉시 처리가 필요하면 운영 봇에 요청하세요.

> 참고: 봇은 **등록된 사용자만** 받습니다. 처음 보는 사람이 보내면 저장하지 않고 "당신의 ID는 …" 안내만 합니다. (보안 기본값)

### 방법 B — 컴퓨터에서 직접

프로젝트 폴더에서:

```bash
python3 scripts/mem.py save "차량 와이퍼 오늘 교체"
python3 scripts/mem.py save "이 도구 써보자 https://github.com/foo/bar"
python3 scripts/mem.py save --file ~/Downloads/영수증.pdf
python3 scripts/mem.py save --file ~/Downloads/스크린샷.png
```

### 태그로 힌트 주기 (선택)

문장 안에 `#태그`를 넣으면 분류·보호에 반영됩니다:

- `#private` → 민감 기록으로 표시(넓게 노출되는 폴더로 안 옮김)
- `#task` `#maintenance` `#음악` 등 → 분류 힌트
- `#노요약` → 링크 자동 요약 대상에서 제외

> **원본은 절대 바뀌지 않습니다.** 캡처한 그대로 `00_Inbox/Raw`에 보관되고, 정리는 "복사본"인 구조화 노트로만 이뤄집니다. 그러니 마음 놓고 던지세요.

## 3. 자동 정리 (Lint)

쌓인 raw 기록을 **구조화 노트**로 바꾸는 과정입니다. 적절한 폴더로 분류하고, 관련 인물/장소/물건(엔티티)·지도(MOC)·날짜까지 정리합니다.

- **자동 실행**: 매일 22:00에 알아서 돕니다. 보통은 신경 쓸 필요 없습니다.
- **직접 실행**: `python3 scripts/mem.py lint`
- **캡처 봇에 요청**: `/lint`(또는 `/정리`) 전송 → **23시 배치**로 처리
- **운영 봇에 요청**: "안 정리된 메모 정리해줘" → **즉시** 처리

정리 시 자동으로 해주는 것들:

- **분류**: 내용을 보고 maintenance/task/product/food_drink/song… 등으로 폴더 배치
- **날짜 보강**: "6/16", "내일", "다음달 5일" 같은 표현을 저장일 기준 **`YY.MM.DD`** 로 환산해 기록에 추가 (예: 26.06.16)
- **중복 제거**: 같은 내용을 여러 번 보내도 구조화 노트는 **하나만** 만듭니다(원본은 다 보존)
- **학습 반영**: 아래 "검토"에서 당신이 정해준 분류 규칙을 적용

## 4. 링크·미디어 자동 요약

### 링크 요약 (웹 페이지 + 유튜브)

URL이 포함된 메모는 **자동으로 내용을 가져와 한국어 요약**을 만들어 옵시디언 노트에 추가합니다.

- **웹 페이지**: 제목·대표 이미지·본문 발췌 + 한국어 3~5줄 요약 + 핵심 포인트
- **유튜브 영상**: 자막을 받아 한국어 "영상 요약"으로 정리 (자막 없는 영상은 제목·설명 기반)
- **처리 시점**: 매일 22:00 링크 추출 → 23:00 AI 한국어 요약 (구독 에이전트 사용)
- **지금 바로 요약**: 운영 봇에 "저장된 링크 요약해줘" 요청

`#노요약` 태그를 붙이면 이 메모는 자동 요약 대상에서 제외됩니다.

### 이미지 OCR + PDF 텍스트 추출

사진·스크린샷·PDF 첨부 메모는 **텍스트 내용이 자동으로 추출**되어 검색 가능해집니다.

- **이미지 (JPG·PNG·HEIC 등)**: OCR로 사진 속 텍스트 인식 (한국어+영어)
- **PDF**: 문서 텍스트 자동 추출 (스캔 PDF는 결과가 없을 수 있음)
- 추출된 원문은 `80_Assets/Extracts/` 에 영구 보관, 노트에 링크로 연결됨
- **처리 시점**: 매일 22:00 추출 → 23:00 AI 한국어 요약
- **음성**: 현재 파일로 저장만 (텍스트 추출은 추후 지원 예정)

> 요약이 붙은 노트는 옵시디언에서 바로 내용을 확인할 수 있고, 검색(/seek)에서도 원문 내용으로 찾을 수 있습니다.

## 5. 애매한 건 검토하기 (Review) — 그리고 시스템이 배웁니다

시스템이 분류를 확신하지 못하면 그 기록을 **`00_Inbox/Review`** 폴더에 모아둡니다.

검토 방법:

```bash
python3 scripts/mem.py review list          # 검토 대기 목록 보기
python3 scripts/mem.py review resolve "review-파일이름.md" --type task --signal "송금"
```

- `--type` : 이 기록의 올바른 분류 (예: task, product, maintenance)
- `--signal` : 그렇게 판단한 **핵심 키워드** (예: "송금"). 짧고 구체적인 단어로.

### ✨ 핵심: 시스템이 학습합니다

같은 `--signal`로 **2번 일관되게** 분류를 정해주면, 그 키워드는 **자동 분류 규칙으로 승격**됩니다. 이후로는 같은 키워드가 들어간 기록을 **Review 없이 자동으로** 올바르게 분류합니다.

- 모순되는 결정(같은 키워드인데 다른 분류)은 자동 적용되지 않고 보류됩니다.
- 배운 규칙 확인/삭제:
  ```bash
  python3 scripts/rules.py list           # 배운 규칙 목록
  python3 scripts/rules.py remove "송금"  # 잘못 배운 규칙 취소
  ```
- 사람이 읽기 쉬운 요약은 볼트의 `90_System/Rules/Learned Rules.md` 에도 자동 생성됩니다.

### 분류가 틀렸다면? — 재분류

이미 정리된 노트의 분류가 틀렸을 때:

```bash
python3 scripts/mem.py reclassify "노트파일경로" --type idea
```

또는 운영 봇에: "이 메모 idea 분류로 바꿔줘"

재분류는 노트를 올바른 폴더로 이동하고, 마커와 위키링크도 자동으로 업데이트됩니다.

## 6. 찾기 (검색)

```bash
python3 scripts/mem.py seek "와이퍼 교체"
python3 scripts/mem.py seek "카페" --type food_drink
python3 scripts/mem.py seek "송금" --since 2026-01
python3 scripts/mem.py seek "프로젝트" --tag 차량 --limit 5
```

- 여러 단어를 넣으면 **더 많이 일치하는 기록이 위로** 정렬됩니다(관련도 순).
- `--type` (분류), `--tag` (태그/해시태그), `--since` (이 날짜 이후, 예 `2026-01`) 로 좁힐 수 있습니다.
- 이미지·PDF 첨부 메모는 **추출된 텍스트 내용으로도 검색** 가능합니다.
- **캡처 봇**: `/seek 검색어`(또는 `/검색 검색어`) → 즉시 키워드 결과.
- **더 깊은 검색**: 운영 봇에 "어제 메모한 식당 찾아줘"처럼 자연어로 요청.

## 7. 요약 보기 / 상태 점검

```bash
python3 scripts/mem.py digest    # raw 몇 개, 처리 몇 개, 분류별 통계
python3 scripts/mem.py doctor    # 도구 설치 상태 점검
```

캡처 봇에서는 `/digest`(`/통계`), `/status`(`/상태`), `/doctor`(`/점검`) 로 같은 정보를 받을 수 있습니다. (`/상태`는 즉시, `/통계`·`/점검`은 약 5분 내)

### 주간 회고 (자동 발송)

매주 1회, 설정된 시간에 **캡처 봇으로 주간 회고 메시지**가 자동으로 전송됩니다:
- 이번 주 저장 건수 및 분류 통계
- 검토 대기 건수, 미처리 링크/미디어 요약 현황

따로 요청하지 않아도 알아서 한 주의 기억 현황을 알려줍니다.

## 9. 알아두면 좋은 것

- **원본(Raw)은 신성합니다.** 시스템은 원본을 절대 수정/삭제하지 않습니다. 정리는 항상 복사본(구조화 노트)으로만 합니다.
- **분류가 틀렸다면?** 운영 봇에 "이 메모 task로 바꿔줘"라고 말하거나, `review resolve`로 올바른 분류 + 키워드를 정해주세요. 두 번 정해주면 다음부턴 자동입니다.
- **민감한 기록**은 `#private`를 붙이세요. 넓게 노출되는 위치로 옮기지 않습니다.
- **날짜는 자동 보강**됩니다. "다음달 5일"처럼 적어도 저장 시점 기준으로 `YY.MM.DD`가 붙습니다.
- **중복 걱정 안 해도 됩니다.** 같은 링크를 또 보내도 정리본은 하나만 생깁니다.
- **링크/미디어 요약은 하루 한 번(23시 배치)** 처리됩니다. 지금 바로 보고 싶으면 운영 봇에 요청하세요.

## 10. 자주 쓰는 명령 치트시트

> 한 화면 요약본은 볼트 `90_System/Guides/명령어 빠른 참조.md` 에도 있습니다.
> **봇 둘:** 저장·사진·PDF = 캡처 봇(`@my_lifelog_memory_bot`) / 검색·정리·요약(즉시) = 운영 봇(`@songbongs_CCC_bot`)

| 하고 싶은 것 | 캡처봇 명령 | 운영봇 / CLI | 처리 시점 |
|---|---|---|---|
| 빠르게 저장 | 그냥 전송 | `mem.py save "내용"` | 즉시 |
| 링크·유튜브 저장 | URL 붙여넣기 전송 | `mem.py save "URL"` | 즉시 저장, 요약은 23시 |
| 사진·PDF 저장 | 파일 전송 | `mem.py save --file ...` | 즉시 저장, OCR은 22:00 |
| 도움말 | `/도움`(`/help`) | — | 즉시 |
| 검색 | `/검색 검색어` | 운영봇 자연어 / `mem.py seek …` | 즉시 |
| 상태 보기 | `/상태`(`/status`) | — | 즉시 |
| 통계·점검 | `/통계` · `/점검` | `mem.py digest` / `doctor` | 약 5분 |
| 정리하기 | `/정리`(`/lint`) | 운영봇 "정리해줘" | 23시 배치 / 즉시 |
| 링크 요약 | `/웹요약`(`/enrich`) | 운영봇 "링크 요약해줘" | 23시 배치 / 즉시 |
| 재분류 | — | 운영봇 "이 메모 idea로" | 즉시 |
| 배운 규칙 | — | `rules.py list/remove` | — |
| 수리 | `/수리`(`/repair`) | — | 23시 배치 |

# mem-seek

Life Memory Vault에서 기억을 찾아 합성 답변을 제공한다.
단순 키워드 검색이 아닌 의미 기반 검색으로 관련 기억 전체를 종합한다.

## 검색 프로토콜 (반드시 이 순서로)

### 1단계: 검색 범위 파악

`docs/vault-index.md`를 읽어서 쿼리에 맞는 폴더를 먼저 특정한다.

```bash
# vault-index.md 참조 후 관련 폴더 결정
# 예: "차량 와이퍼" → 20_Records/Maintenance/ + 40_Entities/Things/
```

### 2단계: MOC 확인

해당 카테고리의 MOC 파일을 읽어서 관련 노트 목록을 파악한다.

```bash
# 예: 70_MOCs/Maintenance-MOC.md 읽기
```

### 3단계: Keyword 검색으로 후보 파일 수집

```bash
python3 scripts/mem.py seek "$ARGUMENTS" --limit 10
```

### 4단계: 후보 파일 전체 내용 로딩

keyword 검색 결과의 파일 + MOC에서 파악한 관련 파일을 Read 툴로 전체 내용 로딩.
Karpathy 원칙: 50K 토큰 이하 볼트에서는 전체 컨텍스트 로딩이 RAG보다 낫다.

### 5단계: 의미 기반 합성 답변

로딩한 파일 내용을 바탕으로:
- 정확한 날짜/수치/이름은 출처 파일에서 직접 인용
- 불확실한 정보는 "불확실"로 표시
- 관련 기억을 추가로 제안 ("이것도 관련 있을 수 있어요")

### 6단계: 선택적 저장 (--save 플래그 또는 사용자 요청 시)

합성 답변이 재사용 가치가 있으면 `00_Inbox/Review/seek-[날짜]-[제목].md`에 저장.

```bash
# 저장된 seek 결과는 이후 동일 주제 검색 시 즉시 활용 가능
```

## 답변 형식

```
🔍 "[검색어]" 관련 기억

[합성 답변 — 구체적 날짜, 이름, 수치 포함]

📄 출처:
- [[파일 경로]] — 핵심 내용 한 줄
- [[파일 경로]] — 핵심 내용 한 줄

💡 관련 기억:
- [[연관 노트]] — 연결 이유
```

## 자연어 트리거

- "찾아줘", "언제였어", "뭐였더라"
- "기억해?", "저장한 거 있어?"
- "차량용품 교체 언제 했지"
- "내가 나중에 써보려고 저장한 서비스"
- "지난번에 간 맛집"

## 검색 결과가 없을 때

1. 검색 쿼리를 한국어/영어 동의어로 바꿔서 재시도
2. 관련 상위 카테고리 폴더 전체 탐색
3. Raw inbox도 확인 (아직 lint 안 된 기억일 수 있음)
4. "해당 기억을 찾을 수 없습니다. Raw inbox에 유사한 항목도 없습니다." 로 명시
